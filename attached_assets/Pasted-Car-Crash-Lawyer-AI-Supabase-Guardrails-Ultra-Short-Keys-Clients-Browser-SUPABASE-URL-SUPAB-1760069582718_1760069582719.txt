Keys/Clients

Browser: SUPABASE_URL + SUPABASE_ANON_KEY.

Server: SUPABASE_URL + SUPABASE_SERVICE_ROLE_KEY (server-only; never logged/exposed).

// server/lib/supabaseAdmin.js
const { createClient } = require('@supabase/supabase-js');
exports.supabaseAdmin = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_SERVICE_ROLE_KEY,{auth:{persistSession:false}});


Tables & RLS

Every client table has: auth_user_id uuid references auth.users(id) + index.

Policies (owner-only):

alter table public.<t> enable row level security;
create policy "<t>_sel" on public.<t> for select using (auth.uid()=auth_user_id);
create policy "<t>_ins" on public.<t> for insert with check (auth.uid()=auth_user_id);
create policy "<t>_upd" on public.<t> for update using (auth.uid()=auth_user_id);


Storage

Buckets private. Upload path must start with UID: ${user.id}/....

RLS uses split_part(name,'/',1)=auth.uid()::text.

Share via server signed URLs only.

Typeform identity (hidden fields)

create_user_id = same as user_id (legacy binder for PDF only; not for auth)

email = user email

product_id = selected plan

auth_code = short-lived server nonce (webhook validation)

Webhook (Typeform → server)

Do / Don’t

✅ Anon key in browser; service role on server.

✅ Owner-only RLS; uploads under ${user.id}/....

❌ Don’t trust create_user_id for auth.

❌ Don’t alter storage.objects or disable RLS.
